# Migrate from `create_react_agent` to `create_agent`

## Context

The project uses `create_react_agent` from `langgraph.prebuilt` in two agents. The newer `create_agent` from `langchain.agents` is the recommended replacement — same invoke API and result format, but adds middleware support and cleaner structured output for future use.

## Files to modify

1. `pyproject.toml` — add `langchain` base package
2. `app/agents/root/__init__.py` — swap import + rename `prompt` → `system_prompt`
3. `app/agents/web_research/__init__.py` — same swap

## Steps

### 1. Add `langchain` dependency to `pyproject.toml`
Add `"langchain>=0.4,<1"` to `dependencies` list (consistent with existing version ranges).
Run `uv sync`.

### 2. Migrate web research sub-agent (`app/agents/web_research/__init__.py`)
- Change import: `from langgraph.prebuilt import create_react_agent` → `from langchain.agents import create_agent`
- Change call: `create_react_agent(...)` → `create_agent(...)`
- Rename param: `prompt=` → `system_prompt=`

### 3. Migrate root agent (`app/agents/root/__init__.py`)
- Same 3 changes as step 2.

### 4. (Optional) Remove `langgraph` from `pyproject.toml`
If no other code imports from `langgraph`, remove it and run `uv sync`.

## What stays the same
- `get_response()` signature and all Slack handler code
- `ChatOpenAI` model instances (keep explicit `api_key` passing)
- System prompts, tool definitions, `@tool` decorator
- `ainvoke` call pattern and result extraction

## Verification
1. `uv sync` — installs cleanly
2. `uv run python -c "from langchain.agents import create_agent"` — import works
3. `uv run python main.py` — starts without errors
4. Send a Slack message triggering both root and web_research agents
5. Verify correct responses and log output
