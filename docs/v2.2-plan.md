# V2.2 — Automated Deployment via GitHub Actions

## Goal

Automate the current manual deployment process (push → SSH → pull → docker compose up) using GitHub Actions.

## Current Process (manual)

1. Push changes to `main` on GitHub
2. SSH into Hetzner VPS as root
3. `cd /opt/apps/pidi && git pull`
4. `docker compose up --build -d`

## Automated Process

A single GitHub Actions workflow triggers on every push to `main`:

1. SSHs into VPS as a dedicated `deploy` user
2. Runs `cd /opt/apps/pidi && git pull && docker compose up --build -d`

Uses `appleboy/ssh-action` for the SSH step.

## Implementation

### File to create

- `.github/workflows/deploy.yml` — the workflow

### GitHub Repository Secrets (Settings → Secrets → Actions)

| Secret       | Value                                      |
|--------------|--------------------------------------------|
| `VPS_HOST`   | VPS IP address or hostname                 |
| `VPS_SSH_KEY` | Private SSH key for the `deploy` user     |

### VPS Setup (one-time, as root)

```bash
# 1. Create deploy user with no password login
adduser --disabled-password --gecos "" deploy

# 2. Add to docker group (so it can run docker compose without sudo)
usermod -aG docker deploy

# 3. Set up SSH key auth
mkdir -p /home/deploy/.ssh
# Option A: copy existing authorized_keys
cp /root/.ssh/authorized_keys /home/deploy/.ssh/authorized_keys
# Option B: generate a dedicated key pair (preferred)
# Run on local machine: ssh-keygen -t ed25519 -C "github-deploy"
# Then paste the public key into /home/deploy/.ssh/authorized_keys
chown -R deploy:deploy /home/deploy/.ssh
chmod 700 /home/deploy/.ssh
chmod 600 /home/deploy/.ssh/authorized_keys

# 4. Give deploy user ownership of the project directory
chown -R deploy:deploy /opt/apps/pidi
```

### Workflow Overview

```
push to main → GitHub Actions → SSH into VPS as deploy → git pull → docker compose up --build -d
```

## Design Decisions

- **No container registry** — build happens on VPS directly, keeping it simple
- **No Docker Swarm** — plain `docker compose`, same as current setup
- **Dedicated `deploy` user** — no root access from CI, limited permissions (docker group only)
- **Dedicated SSH key** — recommended so it can be revoked independently
- **`.env` stays on VPS** — secrets never pass through GitHub Actions (only SSH key)
- **`--build` flag** — ensures image is rebuilt with latest code on every deploy
- **`-d` flag** — detached mode so SSH session doesn't hang
- **Existing healthcheck** — validates the deploy via the compose healthcheck already in place
