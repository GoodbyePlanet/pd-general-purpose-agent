# V2.1 — Multi-Agent Architecture

Refactors the single ReAct agent into a two-agent hierarchy. The root agent handles Slack responses and delegation; a dedicated web research subagent owns all internet-facing tools.

## What's new

- **Root agent** — handles conversation and delegates internet tasks via a `web_research` tool; never calls web tools directly.
- **Web research subagent** — wraps `fetch_url` and `web_search`; invoked as an async tool by the root agent.
- **Separate system prompts** — root agent keeps the full Pidi personality; web research agent is task-focused with no personality.

## Notes

- No new packages or config changes — same env vars, same Slack handlers, same Docker setup.
- New agents live in `app/agents/root/` and `app/agents/web_research/`; `app/agent.py` is removed.
- Adding future subagents (Coda, GitHub, logs) only requires a new `app/agents/<name>/` module and registering it as a tool on the root agent.
