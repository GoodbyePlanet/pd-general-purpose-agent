# V2 — Internet Search + URL Fetching

Adds two tools to the agent: web search via Tavily and URL fetching via WebBaseLoader. The bot can now answer questions about current events and read specific links shared in Slack.

## What's new

- **`web_search` tool** — searches the web using Tavily; returns title + URL + snippet for up to `TAVILY_MAX_RESULTS` results (default: 3). Only enabled when `TAVILY_API_KEY` is set.
- **`fetch_url` tool** — fetches a URL and returns its plain-text content (first 8000 chars). Always available, no API key required. Backed by `WebBaseLoader` + BeautifulSoup.
- **System prompt** — updated to instruct the agent when to use each tool and to never tell the user it lacks browsing capabilities.

## Packages added

- `langchain-community` — `WebBaseLoader` for HTML fetching
- `tavily-python` — Tavily search client
- `beautifulsoup4` + `lxml` — HTML parsing required by `WebBaseLoader`

## Config changes

Two new optional env vars (both have defaults so existing deployments are unaffected):

| Variable | Default | Description |
|---|---|---|
| `TAVILY_API_KEY` | `""` | Tavily API key; leave empty to disable web search |
| `TAVILY_MAX_RESULTS` | `3` | Max search results returned per query |

## Notes

- If `TAVILY_API_KEY` is not set, `web_search` is silently disabled and only `fetch_url` is available. A warning is logged at startup: `"Tavily API key not set — web search disabled."`
- No changes to Slack event handlers, FastAPI, or Docker setup.
